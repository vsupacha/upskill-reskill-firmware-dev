4. Firmware defect → error → impact

Microsoft Zune (portable media player):

  • Defect/bug: โค้ดจาก Freescale ที่เป็น supplier
  • Error: เกิดขึ้นครั้งแรกวันที่ 31 ธันวาคม 2008 (leap year)
  • Impact: อุปกรณ์ freeze เป็นเวลา 1 วัน


BOOL ConvertDays(UINT32 days, SYSTEMTIME* lpTime) {
    int dayofweek, month, year;
    UINT8 *month_tab;
 
    dayofweek = GetDayOfWeek(days); // Calculate current day of the week

    year = ORIGINYEAR;
    while (days > 365) {
        if (IsLeapYear(year)) {
            if (days > 366) {
                days -= 366;
                year += 1;
            }
        } else {
            days -= 365;
            year += 1;
        }
    }
    
    month_tab = (UINT8 *)((IsLeapYear(year)) ? monthtable_leap : monthtable); // Check leap year
 
    for (month=0; month<12; month++) {
        if (days <= month_tab[month])
            break;
        days -= month_tab[month];
    }
    month += 1;
    
    lpTime->wDay = (WORD) days;
    lpTime->wDayOfWeek = (WORD) dayofweek;
    lpTime->wMonth = (WORD) month;
    lpTime->wYear = (WORD) year;
 
    return TRUE;
}

4. Testing != debugging

Testing

  • ตรวจหา defect ในซอฟต์แวร์
    • defect = จุดที่ซอฟต์แวร์ไม่ทำงานตาม requirement
  • requirement → ขั้นตอนและเงื่อนไขการทดสอบ (test case)
  • ผลลัพธ์ของการทดสอบ = test report ที่สรุปผล PASS/FAIL

Debugging

  • ตรวจสอบต้นเหตุของ error ในซอฟต์แวร์
    • error = ความแตกต่างของผลลัพธ์เมื่อพิจารณาจาก requirement
  • การทดสอบที่ FAIL → ตำแหน่งที่ดีบั๊ก
  • ผลลัพธ์ของการดีบั๊ก = ตำแหน่งและวิธีแก้โค้ดที่เป็น defect

  • การ overflow ของ integer
uint8_t u8 = 200;

4. Memory in C

Static

  • จัดสรรสำหรับตัวแปรแบบ global และ static
  • ขนาดถูกตรวจสอบและจองในขั้นตอนคอมไพล์: .data และ .bss
  • ตำแหน่งหน่วยความจำไม่เปลี่ยนแปลง
  • หากไม่ประกาศค่าเริ่มต้น มีค่าเป็น 0

Stack

  • จองชั่วคราว: ตัวแปร local, ส่งผ่านค่าตัวแปร, push/pop รีจิสเตอร์ก่อน branch
  • ขนาดขึ้นกับการประกาศใน linker script
  • พื้นที่หน่วยความจำที่ใช้ร่วมกันทั้งโค้ด
  • มีตำแหน่ง/ค่าเริ่มต้นไม่แน่นอน
  • ความเสี่ยงหลักคือ stack overflow

Heap

  • จองเมื่อเรียกใช้ malloc() และคืนพื้นที่เมื่อเรียกใช้ free()
  • ขนาดขึ้นกับการประกาศใน linker script
  • ความเสี่ยงหลักคือ memory flagmentation และ memory leak

5. On-board/intra-board connectivity

  • UART: full-duplex
    • ขา Tx/Rx → async serial1
    • data rate จำกัด (9600-115200 bps)
  • SPI: full-duplex
    • ขา SCLK/MISO/MOSI/CS → sync serial
    • data rate สูง (10-20 Mbps)
  • I2C: half-duplex
    • ขา SDA/SCL → sync serial
    • data rate ปานกลาง (100-400 kbps)


  1. asynchronous คือไม่มี clock signal เข้าจังหวะ

Controller Area Network (CAN)

  • ISO 11898 standard
  • ระบบสัญญาณแบบ differential signalling
    • CAN transceiver → CANH/CANL pins
    • สายสัญญาณ Twisted 5V
  • เครือข่ายแบบ async serial
    • สื่อสารแบบ broadcast
    • กลไก priority ผ่าน message ID

Local Interconnect Network (LIN)

  • ISO 17987 standard
  • ระบบสัญญาณแบบ single wire (12V/24V)
    • LIN transceiver → LIN pin
  • เครือข่ายแบบ async serial
    • 1 commander / 16 responder

5. CAN bus topology

Components1

  • Node: อุปกรณ์ที่รับ/ส่ง CAN frame เข้า CAN bus
  • Repeater: อุปกรณ์ทำหน้าที่เชื่อมโยงระหว่าง CAN bus ที่เหมือนกัน
  • Bridge: อุปกรณ์ทำหน้าที่เชื่อมโยงระหว่าง CAN bus ที่ต่างกัน
  • Gateway: อุปกรณ์ทำหน้าที่เชื่อมโยงระหว่าง CAN bus กับโครงข่ายอื่น

Error frame กระตุ้นให้ส่ง msg ใหม่ ส่ง 6-bit ติดต่อกัน + 8 recessive bit (error delimiter)
Termination ลดการสะท้อนที่ปลายสาย/ค่า DC ที่ 2.5V ต่อตัวต้านทาน 120 \(\Omega\) ที่ปลายสาย
Cable length ลดปัญหา threshold ของ \(V_{CANH} - V_{CANL}\) สาย < 40 m ที่ 1 Mbps, < 100 m ที่ 500 kbps, …

6. Acceptance filter

Std-ID filter: แฟลก SFT ตั้งโหมด และแฟลก SFEC คุมการส่งข้อมูลไป FIFO

Filters Patterns Examples
Range ID มีค่าระหว่าง SFID1/SFID2 SFID1=0x100, SFID2=0x120 → รับทุก ID ที่อยู่ในช่วง 0x100 ถึง 0x120
Dual ID มีค่าเท่ากับ SFID1 หรือ SFID2 SFID1=0x200, SFID2=0x300 → รับเฉพาะ ID 0x200 และ 0x300 เท่านั้น
Bit mask SFID1 เป็น filter ID และ SFID2 เป็น mask1 SFID1=0x400, SFID2=0x7F0 → รับกลุ่ม ID 0x400 ถึง 0x40F

Ext-ID filter: โหมดและ FIFO ขึ้นกับแฟลก EFEC/EFT และเงื่อนไขคือค่า EFID1/EFID2

  1. เงื่อนไข mask bit: 0 คือไม่สนใจ 1 คือเลือก
